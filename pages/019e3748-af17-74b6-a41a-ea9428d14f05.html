TEMPOC のリリース作業について

GitHub Actions で対象ブランチのPRをマージすると 、バージョンアップ処理を行い、タグを作成、タグによってリリース作業を行います。

バージョンアップ

PRがマージされたら処理が開始します。

  1. version ファイルから発行するバージョンを推定
  2. version,manifest.json を編集
  3. 別ブランチでコミットし、main に自動マージ
  4. タグ打ちを行う

mainに自動マージする動作で、当バージョンアップが再度動作しないようにして欲しい。

バージョンの推定

version ファイルには、バージョンが書いてあるが、

  • 同じバージョンのタグがある場合、マイナーバージョンアップ
  • タグがないバージョンは、その文字列をそのまま発行

リリース作業

タグ打ちで起動

  1. src部分 をzip にする（Chrome Extensionへのリリース）
  2. zipを登録したリリースを発行

参考動作

  • D:\Go\Projects\ego\github\workflows\versionup.yml
  • D:\Go\Projects\ego\github\workflows\release.yml
